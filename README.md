## **Іспит з Python Basic AI**

Група: 53

**Варіант №2:** «Облік продажу книг»

**Проєкт:** Система управління магазином фотокниг (Exam Project)

### **Технологічний стек**

- **Мова програмування:** Python 3.x

- **База даних:** PostgreSQL (реляційна БД)

- **ORM:** SQLAlchemy (для взаємодії між кодом та БД)

### **Структура проєкту**
- **main.py** — точка входу в програму; містить логіку головного меню та основний функціонал.

- **models.py** — опис моделей даних та налаштування зв'язку між таблицями БД і Python-об'єктами.

- **db.py** — конфігурація та створення підключення до бази даних.

- **Photobook_Exam.sql** — SQL-скрипт для розгортання структури БД та її наповнення (містить дані про 50 книг, 10+ співробітників та 230+ записів про продажі).

- **requirements.txt** — перелік зовнішніх бібліотек, необхідних для коректної роботи додатка.

### **Інструкція із запуску проєкту**
**1. Підготовка бази даних**

- Переконайтеся, що у вас встановлено PostgreSQL.

- Створіть нову базу даних (наприклад, photobook_db).

- Виконайте SQL-запити з файлу Photobook_Exam.sql у вашій базі даних для створення таблиць та заповнення їх початковими даними.

**2. Налаштування середовища**

- Створіть віртуальне середовище та активуйте його.

- Встановіть необхідні залежності командою:
Bash
pip install -r requirements.txt

- У файлі db.py вкажіть ваші параметри підключення до бази даних (host, port, user, password, database_name).

**3. Запуск програми**

Виконайте команду:
Bash
python main.py

### **Функціональні можливості**

**Адміністрування**

- **Каталог книг:** додавання, редагування та видалення. Реалізовано "м'яке видалення" (is_deleted) для збереження цілісності історії продажів.

- **Довідник співробітників:** повне керування даними персоналу. При видаленні (звільненні) також використовується is_deleted.

- **Валідація:** передбачено захист від помилкового видалення (контрольні запитання) та перевірку коректності введення (наприклад, формат email або умови CONSTRAINTS на рівні БД).

**Операційна діяльність**

- **Реєстрація продажів:** перевірка наявності книги та співробітника. Можливість оформлення кількох примірників однієї книги (кожен відображається окремим рядком).

- **Ціноутворення:** система пропонує роздрібну ціну за прайсом, але дозволяє вказувати фактичну ціну продажу вручну.

- **Автоматизація:** реєстрація продажу автоматично фіксується за поточною датою.

**Аналітика та звітність**

Програма дозволяє формувати 11 звітів, серед яких:

1. Повна інформація (співробітники, книги, продажі).

2. Фільтрація за часом (на дату, за період).

3. Персональна ефективність (продажі конкретного співробітника).

4. Статистика (найбільш продавана книга, найуспішніший продавець, сумарний прибуток).

5. Бонусні звіти (популярність за авторами та жанрами).

**Додаткові можливості звітів:**

- Експорт результатів вибірок у формат **CSV**.

- Розрахунок консолідованих показників (виручка та прибуток) безпосередньо у звітах.

### **Інтерфейс користувача**

Головне меню розділене на три логічні блоки:

**Адміністратор:** керування асортиментом та кадрами.

**Операційна діяльність:** реєстрація нових операцій та перегляд поточної інформації.

**Аналітика та звітність:** формування розрахункових та вибіркових даних.

Система передбачає зручну навігацію з можливістю повернення в меню та зрозумілу обробку помилок введення з паузами для ознайомлення з повідомленнями.
