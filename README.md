### **Система управління книжковим магазином (Exam Project)**

#### **Технологічний стек:**
Python 3.13

PostgreSQL

SQLAlchemy (Core & Reflection)

python-dotenv 

#### **Структура проєкту:**
Проєкт розділений на модулі для легшої підтримки:

**main.py** - точка входу

**cli.py** - інтерфейс користувача; обробка введення та вивід таблиць, меню користувача

**repositories.py** — усі SQL-запити (CRUD, звіти)

**models.py** — автоматичне зчитування структури БД (Reflection) через autoload_with

**db.py** — безпечне підключення до БД через змінні оточення

**validators.py** — функції для перевірки email, дат та числових значень

**services.py** — сервіс експорту звітів у формат CSV

requirements.txt — мінімальний набір бібліотек (SQLAlchemy, Psycopg 3, Dotenv)

#### **Інструкція із запуску**
1. Підготовка бази даних

Система розроблена для роботи з PostgreSQL.

- Переконайтеся, що сервер БД запущений і створена порожня база даних.
- Для автоматичного створення структури та наповнення системи тестовими даними, виконайте SQL-скрипт Photobook_Exam.sql
Скрипт додасть у вашу базу: 50 книг, 10+ співробітників та 230+ записів про продажі для коректної перевірки звітів.

2. Налаштування змінних середовища (.env)

Налаштуйте зв'язок програми з базою даних:

- Створіть файл `.env` на основі `.env.example`.
- Вкажіть параметри вашого локального сервера PostgreSQL:
  - `DB_USER` - ім'я користувача (наприклад, `postgres`).
  - `DB_PASSWORD` - ваш пароль.
  - `DB_HOST` - адреса сервера (зазвичай `localhost`).
  - `DB_PORT` - порт (стандартний — `5432`).
  - `DB_NAME` - назва створеної бази даних.

**Приклад заповнення:**
  - `DB_USER` = postgres`
  - `DB_PASSWORD`= mysecretpassword.
  - `DB_NAME`= bookstore_db

3. Запуск

\```bash
python main.py
\```

#### **Структура даних**
Програма базується на трьох взаємопов'язаних таблицях:

**Книги (books):** ISBN, назва, автор, жанр, рік випуску, собівартість та роздрібна ціна.

**Співробітники (employees):** картки продавців із контактними даними та статусом активності (is_deleted).

**Продажі (sales):** центральна таблиця, що фіксує фактичну ціну продажу на момент угоди. Це дозволяє змінювати ціни в каталозі без спотворення історичної фінансової звітності.

#### **Логіка валідації та безпеки**
**Цінова політика:** реєстрація нової книги можлива лише за умови, що Ціна > 0 та Собівартість > 0. Додатково перевіряється, щоб ціна продажу не була нижчою за собівартість.

**Валідація кількості:** операції з 0 або від'ємною кількістю книг заблоковані

**Механізм Soft Delete (Архівація):** при видаленні книги чи співробітника запис не зникає фізично, а маркується як is_deleted = True. Це зберігає цілісність звітів за минулі періоди.

**Інтелектуальне відновлення:** якщо користувач намагається додати книгу (за ISBN) або співробітника (за Email), які вже є в архіві, система пропонує миттєво відновити їх у системі одним натисканням.

#### **Функціональні можливості**
1. **Адміністрування (Керування ресурсами)**

- Повний цикл CRUD для книг та персоналу.
- Детальні картки об'єктів та керування архівом.

2. **Операційна діяльність (Продажі та каталог)**

- **Реєстрація продажу:** автоматична перевірка наявності книги та активного статусу продавця (неможливо оформити продаж на архівні позиції).
- **Керування історією:** можливість коригування ціни або видалення помилкових записів.
- **Живий каталог:** перегляд актуального списку книг із підрахунком очікуваного прибутку (маржі) для кожної одиниці.

3. **Аналітика та звітність (11 звітів)**

Звіти формуються через SQL-агрегації для максимальної швидкості роботи:

- **Фільтрація:** продажі за дату, період або конкретного співробітника.

- **ТОП-показники:** бестселери, найкращі продавці, популярні автори та жанри

- **Фінансовий аналіз:** Розрахунок виручки, витрат та чистого прибутку.

- **Експорт:** можливість миттєвого збереження звітів з агрегацією даних (вибірок) у формат CSV

#### **Інтерфейс користувача**
Реалізовано логічне ієрархічне меню для зручної навігації між модулями системи:

* **Адміністрування:** повний цикл роботи з довідниками книг та персоналу, управління архівом (Soft Delete) та відновлення записів.
* **Операційна діяльність:** оформлення нових угод, інтерактивна робота з каталогом та інструменти для корегування історії продажів.
* **Аналітика та звітність:** централізований доступ до 11 типів аналітичних звітів з можливістю миттєвого експорту вибірки даних у формат CSV.
